Key learnings & principles

Google OAuth scope authorization for DriveApp must be triggered manually via a test function run in the Script Editor before the deployed web app can use it. Pattern: add test function → run it → approve permissions → delete test function → redeploy via edit existing deployment.
New deployments generate a new URL (breaking Settings); editing existing deployments preserves the URL — always use the pencil icon.
CRA compliance requirements (payroll RP number, WSIB registration, TD1 forms, T4/T5 deadlines, ROE, T5018 for subcontractors) must be set up before they're needed, not at year-end. This insight came from personal filing experience and shaped the compliance timeline feature.
Cloudflare D1 dashboard may display "Tables: 0" incorrectly — the database and tables are confirmed operational; don't trust the dashboard display alone.


Approach & patterns

Preferred implementation format: One complete file per feature with clearly marked sections, exact search terms to locate insertion points, and explicit line number references where visible. Ready-to-paste code with "where to put it" instructions — no markdown explanation documents.
Editor workflow: GitHub web editor, using Ctrl+F to locate code and selecting line ranges to replace functions.
Session continuity: Caleb values having up-to-date project file snapshots (index.html, Code.gs, google-setup.js) carried forward to avoid re-establishing context.
Feature preservation: Strong preference for incremental improvement without regression — new work should never remove or break existing functionality.
